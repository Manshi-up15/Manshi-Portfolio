◈ How It Works

Rotation Transformation

3D rotation is a linear transformation applied via rotation matrices. Here's what happens inside the GPU at every frame.

01
Y-AXIS ROTATION

Rotating around the Y-axis pivots the object horizontally. The X and Z coordinates transform while Y stays fixed — creating the classic turntable effect.

x' = x·cos(θ) + z·sin(θ)
y' = y
z' = −x·sin(θ) + z·cos(θ)
02
X-AXIS ROTATION

Mouse drag modulates the X-axis rotation — tilting the model toward or away from the viewer. Y and Z coordinates transform while X remains constant.

x' = x
y' = y·cos(θ) − z·sin(θ)
z' = y·sin(θ) + z·cos(θ)
03
DEPTH DISPLACEMENT

Pixel luminance drives vertex displacement along the Z-axis. Bright pixels push outward, dark pixels recede — simulating a true 3D surface from a flat image.

z = luminance(R,G,B) × depth_scale
vertex.z += z
04
MATRIX COMPOSITION

Three.js composes the world matrix from translation, rotation, and scale matrices — applied in that exact order via matrix multiplication on the GPU.

M = T · Ry · Rx · Rz · S
gl_Position = MVP · vertex
