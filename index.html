About Me

I'm Manshi — a UI/UX designer studying Computer Science at SRM Institute. I'm drawn to the gap between how systems actually work and how people experience them. That gap is where I design.

My CS background means I think in systems, not just screens. I can read a developer's constraints and design around them — not against them. I've shipped real designs that got implemented without endless revision cycles.

I care about usability over novelty. An interface that works invisibly is harder to design than one that looks impressive. I'm always chasing the former.

Right now I'm looking for opportunities where I can go deeper — into research, real user feedback, and the kind of iteration that only happens in a product environment.

How I Think
Design thinking
in practice.
01 / Systems

I design for the whole, not the screen.

Before touching Figma, I map user flows and edge cases. A screen without context is decoration. Every interface must be part of a coherent journey.

02 / Constraints

Limits sharpen decisions.

My CS background means I design knowing what's buildable. Technical constraints are creative parameters — this makes developer collaboration genuinely smoother.

03 / Clarity

If it needs explaining, it needs redesigning.

I test every design against one question: would a new user know what to do here without guidance? If no — the interface has more work to do.

04 / Research

Academic rigor applied to design problems.

Work on satellite image processing (quadtree approaches) trained me to structure complex visual data — skills that map directly to data-heavy UI challenges.

05 / Iteration

First ideas are hypotheses, not solutions.

I treat wireframes as questions, not answers. Real design begins after the first version gets challenged — by feedback, edge cases, and reality.

06 / Empathy

The user is never wrong, the design is.

When someone can't figure out an interface, blaming the user is always wrong. I design for the actual human in front of the screen — not the idealized one in my head.
