Definition & Problem Statement
◈ N-Queens — Definition

The N-Queens problem is a classical combinatorics puzzle: place N queens on an N×N chessboard such that no two queens attack each other. A queen attacks any piece sharing its row, column, or either diagonal. The challenge is to find all valid non-attacking arrangements — a foundational problem in constraint satisfaction and backtracking algorithms.

◈ Problem Statement

Given an N×N orbital grid, deploy exactly N satellites — one per row — such that no two satellites occupy the same column or lie on the same diagonal trajectory. Satellites broadcast on identical frequencies; any shared row/column/diagonal causes destructive signal interference, corrupting mission-critical telemetry data.

05
Backtracking & Constraint Pruning
📌
Place
Try column c in current row. Check if safe against all previous queens.
✅
isSafe()
Verify no queen shares same column or diagonal offset |row−r| = |col−c|.
⬇️
Recurse
If safe, recurse to row+1. If all N rows filled → solution found.
↩️
Backtrack
If no valid column in row, undo last queen and try next column above.
✂️
Pruning
Invalid branches cut early — entire subtrees skipped, not just single cells.
⚡ Pruning Effect

Without pruning: brute-force checks all N^N positions. With backtracking + constraint propagation, the search space shrinks dramatically — for N=8, only ~2,057 nodes are visited vs 16 million brute-force permutations. The isSafe() guard prunes entire branches the moment a conflict is detected, before recursing deeper.

function bt(row) { // row = current orbital band
  for c in 0..N: if isSafe(board, row, c): board[row]=c; bt(row+1)
  if row == N: → SOLUTION FOUND  |  else: board[row]=-1; ← BACKTRACK }
04
Complexity
TIME COMPLEXITY
O(N!)

Worst-case explores all N! permutations. Backtracking + pruning reduces average case significantly — practical for N ≤ 15.

SPACE COMPLEXITY
O(N)

Board stored as 1D array of length N (column index per row). Recursion stack depth = N. Highly space-efficient.
